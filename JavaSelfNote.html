Chapter 1 :- Data Types and Variables.

Java Is a Strongly Typed Language

Java is strongly typed language. Indeed, part of Java’s safety and robustness comes from this fact. Let’s see what this means. First, every variable has a type, every expression has a type, and every type is strictly defined. Second, all assignments, whether explicit or via parameter passing in method calls, are checked for type compativlity. There are no automatic coercions or conversion of conflicting types as in some languages. The Java compiler checks all expressions and parameters and parameters to ensure that the types are compativle. Any type mismatches are errors that must be corrected before the compiler will finish compiling the class.

Primitive Data Type

Java defines eight primitive types of data : byte, short, int, long, char, float, double, and boolean. The primitive types are also commonly referred to as simple types.

  • Integers :- This group includes byte, short, int, and long, which are for whole-vlaued signed numbers.
  • Floating-point numbers :- This group includes float and double,which represent numbers with fractinal precision.
  • Characters :- This group includes char, which represents symbols in a character set, like letters and numbers.
  • Boolean :- This group includes boolean, which represents symbols in a character set, like letters and numbers.

You can use these types as-is, or to construct arrays or your own class types. Thus, they form the basic for all other types of data that you can create.

The primitive types represent single values-not complex objects. Although Java is otherwise completely object-oriented, the primitive types are not. They are analogous to the simple types found in most other non-objected languages. The reason for this is efficiency. Making the primitive types into objects would have degraded performance too much.

Integers

byte

The smallest integer type is byte. This is a signed 8-bit type that has a range from –128 to 127. Variables of type byte are especially useful when you’re working with a stream of data from a network or file. int

The most commonly used integer type is int. It is a signed 32-bit type that has a range from –2,147,483,648 to 2,147,483,647. In addition to other uses, variables of type int are commonly employed to control loops and to index arrays. Although you might think that using a byte or short would be more efficient than using an int in situations in which the larger range of an int is not needed, this may not be the case. The reason is that when byte and short values are used in an expression, they are promoted to int when the expression is evaluated. (Type promotion is described later in this chapter.) Therefore, int is often the best choice when an integer is needed

Floating-Point Types

Floating-point numbers, also known as real numbers, are used when evaluating expressions that require fractional precision. For example, calculations such as square root, or transcendentals such as sine and cosine, result in a value whose precision requires a floatingpoint type. Java implements the standard (IEEE–754) set of floating-point types and operators. There are two kinds of floating-point types, float and double, which represent single- and double-precision numbers, respectively. Their width and ranges are shown here:

Name Meomory Size in Bits Approximate Range
double 64 4.9e–324 to 1.8e+308
float 32 1.4e–045 to 3.4e+038

float

The type float specifies a single-precision value that uses 32 bits of storage. Single precision is faster on some processors and takes half as much space as double precision, but will become imprecise when the values are either very large or very small. Variables of type float are useful when you need a fractional component, but don’t require a large degree of precision. For example, float can be useful when representing dollars and cents.

Syntax :- float VariableName;
Example :- float s;

double

Double precision, as denoted by the double keyword, uses 64 bits to store a value. Double precision is actually faster than single precision on some modern processors that have been optimized for high-speed mathematical calculations. All transcendental math functions, such as sin( ), cos( ), and sqrt( ), return double values. When you need to maintain accuracy over manyiterative calculations, or are manipulating large-valued numbers, double is the best choice.

Syntax :- double VariableName;
Example :- double s;

Booleans

Java has a primitive type, called boolean, for logical values. It can have only one of two possible values, true or false. This is the type returned by all relational operators, as in the case of a < b. boolean is also the type required by the conditional expressions that govern the control statements such as if and for loop.

Scope

  • Defination :The scope of a variable defines the region of the program where the variable is accessible. In Java, there are two primary scopes relevant to methods: class scope and method scope.
  • Class Scope : Variables declared at the class level are accessible to all methods within that class. These are called fields or member variables. They persist as long as the class instance exists.
  • Method Scope : Variables declared inside a method are local to that method. This scope begins with the method's opening curly brace { and ends with its closing curly brace }. Method parameters are also part of this scope.
  • Block Scope Within methods, blocks of code defined by curly braces {} create nested scopes. Variables declared in an inner block are not accessible from outer blocks, but outer variables are accessible from inner blocks.

Lifetime

  • Defination :The lifetime of a variable refers to the duration for which the variable exists in memory and holds its value.
  • Method Scope Lifetime: Variables declared within a method are created when the method is invoked and destroyed when the method finishes execution. Thus, they do not retain their values between method calls.
  • Block Scope Lifetime: Variables declared within a block exist only during the execution of that block. They are initialized when the block is entered and destroyed when the block is exited.
